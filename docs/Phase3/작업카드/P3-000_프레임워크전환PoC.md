# P3-000 프레임워크 전환 PoC (Viser → PyVista+Trame)

## 목적
Viser의 상호작용 한계가 실제 라벨링 UX 병목인지 검증하고, PyVista+Trame 전환 여부를 결정한다.

## 배경
Phase 2에서 확인된 Viser 한계:
- painting 중 orbit 회전 불가 (pointer event가 orbit 차단)
- 드래그 연속 페인팅 불가 (click 1회씩만)
- modifier 키(Ctrl/Shift) 감지 불가
- mesh 색상 in-place 업데이트 불가 (remove + re-add, 깜빡임)

PyVista+Trame 비교 기준:
- VTK 기반, vertex picking / rubber-band selection / drag 이벤트 지원
- 웹 UI 유지 가능 (Trame = Vue.js + vtk.js)
- orbit과 painting 동시 가능 (마우스 버튼 분리)
