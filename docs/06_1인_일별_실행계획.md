# 1인 일별 실행계획 (4주, 03/26 ~ 04/24)

## 전제 조건
- 개발자: 1인 (고박사)
- 환경: Viser / Open3D 미설치, YCB asset 미확보
- 시작일: 2026-03-26 (오늘)
- 기술 스택: Python + Viser (웹 UI) + Open3D (백엔드) + JSON (저장)

## 핵심 원칙
- **매일 끝에 "동작하는 무언가"를 남긴다** — 커밋 가능한 상태를 유지
- **수직 슬라이스** — UI→백엔드→저장을 가로지르는 하나의 기능을 끝까지 뚫는다
- **시간 박스** — 하나의 작업에 반나절(4h) 이상 막히면 우회로를 택한다
- **A/B/C/D 역할을 직렬화** — 하루 안에서 UI→데이터→기준정의를 순서대로 처리

## 1인 개발 시간 관리 팁
- **오전 집중 블록 (3~4h)**: 새로운 기능 구현
- **오후 마무리 블록 (2~3h)**: 테스트, 연결, 커밋
- **일일 커밋 목표**: 최소 1개 의미 있는 커밋
- **막힘 규칙**: 4시간 이상 한 가지에 막히면 → 우회로 선택 → 다음 날 재도전
- **주말**: 선택적 진행 (번아웃 방지), 버퍼로 활용
