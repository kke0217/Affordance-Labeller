# 데이터 스키마 초안

## 목적
이번 달 내부 JSON v0.1의 필수 필드와 확장 대비 필드를 정리한다. 목표는 완전한 표준 포맷 구현이 아니라, 추후 RLDS / LeRobot 변환이 가능한 구조를 확보하는 것이다.

## 최상위 필드
- object_id
- input_type
- asset_version
- canonical_frame
- parts
- affordances
- contact_region_masks
- candidate_poses
- annotator
- review_status
- updated_at

## 필드 설명
### object_id
예: ycb_025_mug

### input_type
예: rgb, depth, rgbd, pointcloud, mesh

### canonical_frame
- origin
- axis_x
- axis_y
- axis_z

### parts
각 part는 다음 속성을 가짐
- part_id
- name
- visible
- comment

예시 part name
- body
- handle
- rim
- interior

### affordances
각 affordance는 다음 속성을 가짐
- affordance_id
- label
- part_ref
- semantic_tags
- source_type
- comment

권장 affordance label
- graspable
- pour_support
- handover_region
- non_affordance

### contact_region_masks
각 mask는 다음 속성을 가짐
- mask_id
- mask_type
- part_ref
- patch_a
- patch_b
- comment

권장 mask type
- handle_pinch
- body_power
- rim_control

### candidate_poses
각 pose는 다음 속성을 가짐
- pose_id
- name
- translation
- rotation_xyzw
- linked_affordance_id
- linked_mask_id
- semantic_tags
- grasp_type
- confidence
- approved
- comment

### annotator
라벨 작성자

### review_status
- draft
- reviewed

### updated_at
ISO timestamp

---

## 확장 대비 필드
- task_label
- hand_role
- grasp_type
- source_type
  - manual / assisted / auto
- confidence
- downstream_export_ready

## 설계 원칙
- 지금은 간단하게 유지한다
- 그러나 나중에 RLDS / LeRobot로 매핑할 수 있어야 한다
- 필드 이름은 중간에 자주 바꾸지 않는다
- pose / affordance / mask 연결 관계를 명확히 유지한다

## 샘플 객체
- 1차 기준 객체: YCB mug
- 확장 대상: YCB 추가 객체, PartNet-Mobility articulated object
