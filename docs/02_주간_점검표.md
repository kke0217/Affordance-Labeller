# 주간 점검표

## 운영 원칙
- 매주 5분 데모를 우선한다
- 완료/미완료를 기능 기준으로 판단한다
- 새 기능 추가보다 기존 기능 안정화를 우선한다
- 자동화·확장 논의는 게이트 통과 후로 미룬다

---

## 1주차 점검표 — 기술 스택 고정과 입력 루프 확보
### 핵심 질문
YCB mug를 불러와 저장·재로드할 수 있는가?

### 반드시 확인할 항목
- [ ] Viser 개발환경 구동 성공
- [ ] YCB mug asset 로드 성공
- [ ] RGB-D 또는 mesh 뷰 표시 성공
- [ ] object_id / input_type 표시 성공
- [ ] canonical frame 임시 저장 성공
- [ ] label JSON skeleton 저장 성공
- [ ] 저장 파일 재로드 성공

### 시연 항목
- mug 로드 화면 시연
- save → reload 시연

### 실패 신호
- [ ] 3일 이상 viewer가 안 뜸
- [ ] 좌표계 기준이 팀 내 합의 안 됨
- [ ] JSON 구조가 계속 바뀜

## 2주차 점검표 — affordance와 part 라벨링
### 핵심 질문
연구원이 실제로 affordance와 tag를 입력할 수 있는가?

### 반드시 확인할 항목
- [ ] body / handle / rim / interior 표시 성공
- [ ] part 수동 수정 가능
- [ ] affordance brush 또는 polygon 구현
- [ ] affordance class selector 동작
- [ ] semantic tag selector 동작
- [ ] 색상 오버레이 표시 성공
- [ ] affordance 저장 / 재로드 성공

### 권장 최소 class
- graspable
- pour_support
- handover_region
- non_affordance

### 권장 최소 tag
- pick_up
- pour_ready
- handover_ready
- reposition_only

### 시연 항목
- handle graspable 라벨 시연
- semantic tag 부여 및 저장 시연

### 실패 신호
- [ ] class/tag 체계가 자꾸 바뀜
- [ ] 편집 UI가 너무 복잡함
- [ ] 저장 후 재로드 시 라벨이 깨짐

## 3주차 점검표 — contact mask와 candidate pose
### 핵심 질문
contact mask와 6D pose를 저장 가능한가?

### 반드시 확인할 항목
- [ ] contact patch A/B 구조 구현
- [ ] mask type 정의 완료
- [ ] candidate pose 리스트 UI 표시
- [ ] pose 추가 / 삭제 / 선택 가능
- [ ] 6D pose 축 시각화 가능
- [ ] pose-affordance-mask 연결 저장 가능
- [ ] JSON 재로드 시 pose 복원 가능

### 권장 최소 mask type
- handle_pinch
- body_power
- rim_control

### 시연 항목
- handle_pinch mask 생성 시연
- pose 1개 추가 및 좌표축 표시 시연

### 실패 신호
- [ ] 6D pose 입력 방식이 너무 복잡함
- [ ] affordance / mask / pose 연결 구조가 깨짐
- [ ] 저장은 되는데 재로드가 안 됨

## 4주차 점검표 — 검수, 안정화, 데모 패키징
### 핵심 질문
연구원이 라벨을 만들고 검토하고 시연할 수 있는가?

### 반드시 확인할 항목
- [ ] review_status 도입
- [ ] annotator / updated_at 기록
- [ ] 필수 필드 validation 구현
- [ ] 저장 실패 / 누락 경고 표시
- [ ] 샘플 1~3개 정리
- [ ] 데모 시나리오 문서화
- [ ] 5분 이내 데모 가능

### 선택 항목
- [ ] 단순 자동 제안 PoC 1개

### 시연 항목
- 전체 라벨 생성 루프 시연
- 저장 → 재로드 → review_status 확인 시연

### 실패 신호
- [ ] 마지막 주에 새 기능 추가 욕심이 생김
- [ ] 포맷 수정 반복
- [ ] 기본 기능보다 자동화 PoC가 우선되고 있음
